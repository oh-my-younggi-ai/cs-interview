# cs-interview

면접 대비 CS 지식을 **직접 쌓고(wiki), 그 지식으로 스스로를 검증하는(interviewer)** Claude Code 시스템.
지식 베이스가 좋아질수록 면접관의 채점이 정확해지는 선순환을 만든다.

## 구성

| 구성요소 | 역할 |
|---|---|
| **cs-wiki** (스킬) | CS 개념을 OKF 마크다운 위키로 정리·교차링크·점검 |
| **cs-interviewer** (스킬) | 위키를 **정답 기준(oracle)** 으로 AI 면접관이 모의 면접·채점·기록 |
| **[knowledge/](./knowledge/)** | 공유 지식 베이스 — 면접관이 문제를 내는 실제 CS 개념 페이지 (JPA·네트워크·OS·DB, 23p) |

```
cs-interview/
├── knowledge/    ← 지식 베이스 (공유·공개, 계속 추가)
├── skills/       ← 엔진 (cs-wiki, cs-interviewer)
├── benchmarks/   ← 스킬 개선 측정 기록
└── records/      ← 면접 기록·오답노트 (로컬 전용 — gitignore, 공개 안 됨)
```

> **공개 원칙:** `knowledge/`(면접관이 낼 지식)는 누구나 쓰도록 공개한다.
> 개인 **면접 기록·오답노트**(점수·약점·답변 원문)는 각자 로컬에만 남기고 **공개하지 않는다.**

## 어떻게 동작하나

```
        ┌──────────────── 정답 기준(읽기 전용) ─────────────────┐
        │                                                      ▼
  [ cs-wiki ] ──정리/갱신/점검──▶  knowledge/  ◀──출제·채점──  [ cs-interviewer ]
                                                                   │
                              약점 가중 출제  ◀──mastery.md──── 면접 기록(로컬·비공개)
```

1. **지식 구축** — cs-wiki 로 개념을 OKF 페이지로 쌓는다. 각 페이지엔 면접관이 쓸 `핵심 개념 / 단골 질문 / 헷갈리는 점 / 관련 개념` 섹션과 `difficulty`·`frequency` 메타가 들어간다.
2. **모의 면접** — cs-interviewer 가 `knowledge/`를 **읽기 전용** 오라클로 삼아, 약점 개념을 우선 + 전체 랜덤 혼합으로 출제. 답변에서 키워드를 뽑아 **꼬리 질문**(개념당 최대 2)으로 깊이 판다.
3. **기록·피드백** — 세션이 끝나면 채점·약점을 개인 기록에 남기고, `mastery.md`가 다음 세션 **출제 가중치**가 된다. 면접이 점점 나에게 맞춰진다.

## 사용법

```
# 지식 쌓기 (cs-wiki) — 3가지 방식
"JPA 더티 체킹 위키에 정리해줘"       # ① 직접: 주제 주면 바로 정리
"N+1이랑 페치조인 얘기하며 정리하자"   # ② 대화 탐색: 대화 → 없는 개념만 정리
"면접 대비 뭐 정리하지?"             # ③ 추천: 위키에 없는 고빈출 주제 추천
"위키 점검해줘"                     # 점검: 모순·고아·누락 리포트

# 모의 면접 (cs-interviewer)
"CS 모의 면접 보자"
"네트워크 위주로 면접 연습하자"
"꼬리질문 받고 싶어"
```

### 지식 축적 3가지 방식 (cs-wiki)

| 입력 | 모드 | 동작 |
|---|---|---|
| 명확한 개념 | **직접** | 바로 정리 |
| 주제만 / "얘기하며 정리하자" | **대화 탐색** | 대화 → 키워드 추출 → 위키에 **없는 것만** 승인 후 정리 |
| "뭐 정리하지?" | **추천** | 위키에 **없는** 고빈출 CS 주제를 이유와 함께 추천 |

## 결과물 (무엇이 나오나)

**1) 지식 페이지** — `knowledge/<주제>/<개념>.md` (OKF)
```yaml
---
type: CS Concept
title: 더티 체킹
difficulty: medium
frequency: high
---
# 핵심 개념 / # 면접 단골 질문 / # 헷갈리는 점 / # 관련 개념
```

**2) 면접 세션 보고서** *(개인 기록 — 로컬 저장, 비공개)* — 대화를 **요약 없이 원문 그대로** 남기고, 틀린 답 **바로 아래 `✍️ 첨삭`**, 개념별 0~100점 + 종합 점수, 끝에 **📚 공부 추천**.
```markdown
## network/tcp-vs-udp — 평가: 양호 (85/100)
> **면접관 [메인]:** {질문 원문}
> **지원자:** {답변 원문}
**✍️ 첨삭:** {틀리거나 빠진 부분}
```

**3) 숙련도 `mastery.md`** — 개념별 시도·최근 점수·약점(✓) 집계 → 다음 출제 가중치.
**4) 로그 `log.md`** — 세션당 한 줄, 최신이 위.

## 지식 베이스 (`knowledge/`) 규칙 — OKF

- **개념 1개 = 파일 1개.** 주제 디렉터리(`network/`, `jpa/` …)로 묶는다. 파일명 `kebab-case`.
- **frontmatter 필수는 `type: CS Concept`.** `difficulty`/`frequency`는 면접관 출제 우선순위에 쓰이니 채운다.
- **교차링크는 bundle-relative**(`/network/three-way-handshake.md`). 대상 페이지가 아직 없어도 링크는 건다("아직 안 쓴 지식").
- **각 디렉터리 `index.md`** = 카탈로그(설명은 페이지 `description`에서), **루트 `log.md`** = 변경 이력(최신 위).

## 지식 업데이트 · 정리 루틴

새 개념을 넣거나 고칠 때 **매번 아래를 지킨다** (cs-wiki 가 자동으로 수행):

**A. 개념 추가/갱신 (ingest)**
1. `concept-template` 형식으로 페이지 작성 — 면접용 4개 섹션(핵심/질문/함정/관련)을 채운다.
2. **덮어쓰지 않는다.** 기존 페이지는 구조 보존하며 **병합/갱신**.
3. **자체 점검** — 방금 쓴 페이지의 frontmatter·필수 섹션·교차링크가 규칙에 맞는지 확인.
4. **index.md · log.md 갱신** (필수). 새 개념이면 관련 *기존* 페이지의 `# 관련 개념`에 **역링크** 한 줄 추가.

**B. 위키 점검 (lint) — "위키 점검해줘"**
1. 전체를 훑어 **모순 / 낡은 서술 / 고아 페이지 / 누락 개념 / 끊긴 구조**를 찾는다.
2. **자동 수정하지 않고 분류 리포트로 보고** → 사용자 승인 후에만 고친다.
3. 처리 내용은 `log.md`에 `* **점검**: …` 로 남긴다.

## 핵심 불변식 (INVARIANT)

- **지식과 기록의 분리** — 위키엔 검증된 지식만. 점수·답변·약점 같은 기록은 절대 위키에 안 넣는다(개인 records 소관).
- **면접관은 위키를 읽기만 한다** — 수정 금지. 위키에 없는 개념은 "정답 기준 미검증"으로 표시하고 정리를 제안만.
- **답변 대필 금지** — 질문 직후 턴 종료, 실제 사용자 입력 뒤에만 진행.

## 설치 / 설정

```bash
# 스킬 설치
cp -R skills/cs-wiki skills/cs-interviewer ~/.claude/skills/
```
- 이 `knowledge/`를 자신의 위키로 쓰려면, 스킬의 `wiki_root`를 이 폴더로 지정하거나 원하는 위치로 복사한다.
- **면접 기록(`records_root`)은 개인 로컬 경로**로 두고 공개 저장소에 넣지 않는다.
